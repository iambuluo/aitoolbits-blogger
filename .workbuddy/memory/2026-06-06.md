## 2026-06-06

### 修复 workflow 超时导致博客未更新

**问题：** 用户报告昨天博客只更新了 1 篇，GitHub Actions workflow 显示 Cancelled。

**根因：** `publish-article.yml` 的 Random delay 步骤最大延迟为 2 小时 (`MAX_DELAY=$((2 * 3600))`)，而 job 总 timeout 只有 60 分钟。导致大多数 schedule 触发的运行还没开始发布就被强制取消。

**修复：**
- `publish-article.yml`: random delay 2h -> 20min，步骤增加 `timeout-minutes: 20`
- `publish-trending.yml`: random delay 1h -> 15min，步骤增加 `timeout-minutes: 15`

**提交：** `7b7e001`

### 待验证
用户需要去 GitHub Actions 页面手动触发两个 workflow 验证修复是否生效。
