# 2026-06-14 Anti-Duplicate Fix Deployment

## Problem
- `published_urls.json` 完全为空，防重复机制形同虚设
- `github_trending.py` 的 `published_repos.json` 也不存在
- 每次生成文章都用随机 star 数，导致 ECC 等文章反复出现
- DeepSeek 的 `get_random_topic()` 每次从 topics.py 选相同选题，但标题因 star 数差异而绕过检查

## Fix
1. 修改 `published_urls.py`：添加 title normalization（去除 star count 差异），改进 `_normalize_title()` 正则匹配
2. 修改 `generate_article.py`：在 `generate_article()` 和 `generate_article_from_topic()` 中都加入 pre-flight 重复检查
3. 修改 `publish_to_blogger.py`：发布成功后自动记录 URL 到 `published_urls.json`
4. 新增 `sync_existing_articles.py`：通过 Blogger API 拉取已有文章填充数据库
5. 新增 `check_gsc_indexing.py` + `gsc-monitor.yml`：GSC 每3天检查一次，Server酱推送到微信
6. 禁用 `publish-trending.yml` → `_disabled-publish-trending.yml`
