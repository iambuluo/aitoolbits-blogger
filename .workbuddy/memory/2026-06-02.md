## Google OAuth Token 反复过期问题修复

### 根因
Google Cloud Console 的 OAuth 同意屏幕处于「测试」状态，refresh token 7天后自动过期。

### 解决方案

**方案A（根本修复）**：在 Google Cloud Console 将 OAuth 应用发布到「生产」状态
- 按钮位置：https://console.cloud.google.com/apis/credentials/consent → 「发布应用」
- 发布后 refresh token 永久有效

**方案B（代码防护）**：代码层面新增 token 轮换自动保存
- `get_access_token()` → 新增 `on_new_refresh` 回调参数
- 当 Google 返回新 refresh_token 时自动触发保存
- 新增 `_try_auto_save_refresh_token()` — 三级保存策略：
  1. 更新 GitHub Secret（需要 GH_PAT 环境变量）
  2. 保存到本地 blogger_tokens.json
  3. 打印到日志供手动更新
- 两个 workflow 都新增 `GITHUB_TOKEN: ${{ secrets.GH_PAT || '' }}` 环境变量

### 修改文件
- `scripts/publish_to_blogger.py` — 新增 token 轮换检测 + 自动保存
- `scripts/publish_trending.py` — 传入 token 轮换回调
- `.github/workflows/publish-article.yml` — 添加 GH_PAT 可选 secret
- `.github/workflows/publish-trending.yml` — 添加 GH_PAT 可选 secret

### 用户操作
1. 可选：在 Google Cloud Console 发布 OAuth 应用（一劳永逸）
2. 可选：在 GitHub Secrets 添加 `GH_PAT` = PAT token，启用 token 自动保存

## OAuth 应用发布生产模式 + 新 Token 获取

### 过程
- Google Cloud Console UI 已变更：发布按钮从旧「OAuth consent screen」移到新「Google Auth Platform → Audience」标签页
- 找到位置后成功发布到生产模式（正式版）
- OAuth 回调从 `urn:ietf:wg:oauth:2.0:oob`（已废弃）改为 `http://localhost`
- 重新获取了生产模式的 refresh token

### 结果
- ✅ 应用发布状态：正式版（Production）
- ✅ 新 refresh token 已写入本地 `blogger_tokens.json`
- ⚠️ PAT 缺少 secrets:write 权限，无法用 API 更新 GitHub Secret
- ⚠️ 用户需手动在 GitHub Settings → Secrets → Actions 更新 `BLOGGER_REFRESH_TOKEN`

### 清理
- 删除临时敏感文件 `new_token_result.json`、`oauth_auto.py`
- 更新 `scripts/refresh_blogger_token.py`、`setup_oauth.py`、`publish_to_blogger.py` 中的 redirect_uri 为 `http://localhost`

### 代码文件变更
- `scripts/refresh_blogger_token.py` — redirect_uri 改为 http://localhost
- `scripts/setup_oauth.py` — redirect_uri 改为 http://localhost
- `scripts/publish_to_blogger.py` — 两处 redirect_uri 改为 http://localhost（replace_all）
