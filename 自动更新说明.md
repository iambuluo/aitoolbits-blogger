# 自动更新说明

## 1. 是否全部自动运行？

**是的，完全自动。**

两个 workflow 都配置了 `cron` 自动触发：

| 类型 | Workflow 文件 | 频率 | 北京时间 |
|---|---|---|---|
| 普通文章 | `publish-article.yml` | 每天 3 篇 | 08:17 / 14:42 / 21:08 |
| Trending 文章 | `publish-trending.yml` | 每天 1 篇 | 10:00 |

你不需要做任何操作，GitHub Actions 会自动执行，文章会直接发布到博客 [aitoolbits.blogspot.com](https://aitoolbits.blogspot.com)。

---

## 2. 万一没自动跑，如何手动触发？

### 方法一：GitHub 网页手动触发（最简单，推荐）

1. 打开 https://github.com/iambuluo/aitoolbits-blogger/actions
2. 点左侧列表中的 **"Publish Articles (Auto 3x Daily)"** 或 **"Publish Trending AI Repo (Daily)"**
3. 点右侧 **"Run workflow"** 按钮
4. 选择分支（默认 `main`），点绿色的 **"Run workflow"** 确认

> 触发后等待 2-5 分钟，文章就会出现在博客上。

---

### 方法二：本地命令行触发（需要安装 GitHub CLI）

先安装 `gh` CLI，然后运行：

```bash
# 触发普通文章（1篇）
gh workflow run "Publish Articles (Auto 3x Daily)" --field article_count=1

# 触发 Trending 文章（1篇）
gh workflow run "Publish Trending AI Repo (Daily)"
```

---

### 方法三：本地直接跑 Python 脚本（完全本地控制）

适合调试和测试，需要本地配置好环境变量。

**前置条件：**
- 项目目录下有 `scripts/.env` 文件，包含 `DEEPSEEK_API_KEY`、`BLOGGER_*` 等密钥
- 或者已通过 `python setup_github_secrets.py` 配置好

**运行命令：**

```bash
cd D:\小程序\aitoolbits-blogger\scripts

# 发 1 篇普通文章（本地测试，发为草稿）
python publish_to_blogger.py 1 --draft

# 发 1 篇普通文章（正式发布）
python publish_to_blogger.py 1

# 发 1 篇 Trending 文章（正式发布，强制刷新数据源）
python publish_trending.py 1 --refresh

# 查看可用的 Trending 项目（不发布）
python publish_trending.py --list
```

## 3. 快速排查指南

| 现象 | 可能原因 | 解决方法 |
|---|---|---|
| 博客几天没更新 | GitHub Actions 被禁用或失败 | 去 Actions 页面查看最近运行记录 |
| Actions 显示 "cancelled" | workflow 超时（之前是 15 分钟，现已改为 180 分钟） | 手动重新触发一次 |
| 文章发布了但没有图片 | Pexels API Key 失效 | 已改为 Unsplash 免费图片 fallback，应自动修复 |
| Trending 文章重复发了同一个项目 | `data/published_repos.json` 被重置 | 检查该文件是否存在于仓库中 |
