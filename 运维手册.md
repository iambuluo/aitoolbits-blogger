## 二、Refresh Token 机制说明

### 当前状态

Google OAuth 应用已发布为**正式版（Production）**，refresh token **理论上永久有效**，不会再 7 天过期。

### 为什么需要 Refresh Token？

- Google 的 access_token 只有 1 小时有效期
- 每次运行工作流时，代码用 refresh_token 去换一个新的 access_token
- 如果 refresh_token 过期，整个系统就停摆了

### 代码自动防护

`publish_to_blogger.py` 内置了 token 轮换检测：

1. 每次获取 access_token 时，检查 Google 是否返回了新 refresh_token
2. 如果返回了，自动尝试保存到 GitHub Secret（需要 `GH_PAT`）
3. 如果 GitHub 保存失败，降级保存到本地 `blogger_tokens.json`
4. 如果本地也失败，打印到日志供手动更新

### Google OAuth 管理入口

```
Google Cloud Console → APIs & Services → Google Auth Platform → Audience 标签页
https://console.cloud.google.com/apis/
```

## 三、故障排查

### 3.1 发现博客没更新

1. 打开 https://github.com/iambuluo/aitoolbits-blogger/actions
2. 查看最近的 workflow run 状态
3. 点击失败的 run，展开日志查看错误信息

### 3.2 常见错误及处理

| 错误信息 | 含义 | 处理方式 |
|---------|------|----------|
| `HTTP 400: invalid_grant` | Refresh token 过期/无效 | 按下方"手动刷新 Token"流程操作 |
| `HTTP 401: unauthorized` | Access token 过期 | 系统会自动重试，一般不需要处理 |
| `HTTP 403: access_denied` | 权限不足 | 检查 Blogger API 是否启用 |
| `DEEPSEEK_API_KEY` 相关错误 | API Key 无效或额度不足 | 去 https://platform.deepseek.com 检查 |
| `PEXELS_API_KEY` 相关错误 | API Key 无效 | 去 https://www.pexels.com/api/ 检查 |

## 四、手动刷新 Token（最关键的救命操作）

### 前提条件

- 你的电脑在 `D:\小程序\aitoolbits-blogger` 目录下有代码
- 有 Python 3 环境
- 能访问 Google 账号（可能需要科学上网）

### 方法一：运行自动脚本（推荐）

```powershell
# 1. 进入项目目录
cd D:\小程序\aitoolbits-blogger

# 2. 设置 GitHub Token（用于自动推送 Secret）
$env:GITHUB_TOKEN = "你的 GitHub Personal Access Token"

# 3. 运行刷新脚本
python scripts/refresh_blogger_token.py
```

脚本会自动：
1. 打开浏览器让你授权 Google 账号
2. 自动捕获授权码
3. 换取新的 refresh_token
4. 更新本地 `blogger_tokens.json`
5. 推送到 GitHub Actions Secret

### 方法二：手动操作（如果脚本跑不了）

#### 第 1 步：生成授权链接

在浏览器打开（把 CLIENT_ID 换成你的实际值）：

```
https://accounts.google.com/o/oauth2/v2/auth?client_id=343347222757-rq1dke6pum0a65sj58rp791gn17iilko.apps.googleusercontent.com&redirect_uri=http://localhost&response_type=code&scope=https://www.googleapis.com/auth/blogger&access_type=offline&prompt=consent
```

> CLIENT_ID 在 `blogger_tokens.json` 中，或者在 GitHub Secrets 里查。

#### 第 2 步：获取授权码

点"允许"后，浏览器会跳转到 `http://localhost/?code=...`（页面打不开是正常的）。**复制地址栏完整 URL**。

#### 第 3 步：换取 Token

把 URL 发给 WorkBuddy AI（我），我会帮你：
- 提取 code
- 调用 Google API 换取新 refresh_token
- 更新本地文件和 GitHub Secret

#### 第 4 步：更新 GitHub Secret

如果 AI 无法自动推送 GitHub Secret（PAT 权限不够）：

1. 打开 https://github.com/iambuluo/aitoolbits-blogger/settings/secrets/actions
2. 找到 `BLOGGER_REFRESH_TOKEN`
3. 点「Edit」→ 粘贴新 token →「Update secret」
